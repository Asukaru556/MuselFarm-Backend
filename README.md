# 🦪 Бэкенд Системы Управления Мидийной Фермой (NestJS + Prisma 7 + PostgreSQL)

Данный проект представляет собой REST API для автоматизации и мониторинга мидийной фермы: управление инфраструктурой (фермы, участки, канатные линии), учет партий мидий, сбор телеметрии с датчиков (температура, соленость, кислород) и ведение журналов биологических наблюдений.

---

## 🐋 Вариант 1. Быстрый запуск через Docker (Рекомендуемый)

Для этого варианта вам не нужно устанавливать Node.js, PostgreSQL или настраивать базы данных вручную. Все зависимости развернутся автоматически.

### Шаг 1. Клонирование проекта
Откройте терминал, перейдите в рабочую папку и скачайте код:
```bash
git clone https://github.com/Asukaru556/MuselFarm-Backend.git
cd MuselFarm-Backend
```

### Шаг 2. Создание файла конфигурации окружения
В корневой папке проекта создайте новый текстовый файл и назовите его строго: **`.env`**  
Откройте его и вставьте следующие строки:
```text
DATABASE_URL="postgresql://postgres:root@localhost:5432/mussel_farm_db?schema=public"
JWT_SECRET="super-secret-key-change-me-in-production"
```

### Шаг 3. Запуск одной командой
Убедитесь, что программа **Docker Desktop** запущена на вашем компьютере. В терминале внутри папки проекта введите:
```bash
docker-compose up --build
```
*Docker автоматически скачает PostgreSQL 15, установит Node.js зависимости, скомпилирует NestJS-проект и применит все 8 миграций структуры таблиц.*

---

## 🔌 Вариант 2. Локальный запуск (Без Docker)

У вас должны быть установлены **Node.js** (v20 или v22) и **PostgreSQL СУБД** (v15+).

1. Создайте в **pgAdmin** или консоли `psql` пустую базу данных (например, `mussel_farm_db`).
2. Создайте файл **`.env`** в корне проекта и укажите доступы к вашей локальной СУБД:
   ```text
   DATABASE_URL="postgresql://ЛОГИН_ОТ_PG:ПАРОЛЬ_ОТ_PG@localhost:5432/mussel_farm_db?schema=public"
   JWT_SECRET="super-secret-key-change-me-in-production"
   ```
3. Установите библиотеки и накатите структуру таблиц командами:
   ```bash
   npm install
   npx prisma migrate dev --name init_db
   ```
4. Запустите сервер бэкенда:
   ```bash
   npm run start:dev
   ```

### Как тестировать эндпоинты напрямую из браузера (Swagger UI)

Перед началом тестирования обязательно выполните **Шаг 1** и **Шаг 2**, чтобы получить права доступа, иначе СУБД будет отклонять запросы со статусом `401 Unauthorized` [INDEX].

---

#### 🔐 Шаг 1. Регистрация и получение доступа
1. Перейдите в браузере на страницу: `http://localhost:3000/api/docs`
2. Разверните блок **`Auth`** и выберите **`POST /api/auth/register`**.
3. Нажмите кнопку **"Try it out"** и отправьте JSON для создания администратора:
   ```json
   {
     "login": "admin_chief",
     "password": "password123",
     "full_name": "Алексей Николаевич Командир",
     "email": "admin@musselfarm.ru",
     "role": "admin"
   }
   ```
4. Нажмите **"Execute"** (ожидаемый ответ: `201 Created`).
5. Перейдите к роуту **`POST /api/auth/login`**, нажмите **"Try it out"**, введите логин `admin_chief` и пароль `password123`. Нажмите **"Execute"**.
6. Из JSON-ответа скопируйте текстовое значение строки `accessToken` (строго без кавычек).
7. Поднимитесь в самый верх страницы Swagger, нажмите большую синюю кнопку **"Authorize"**, вставьте скопированный токен в поле **Value** и нажмите **Authorize**, после чего закройте окно [INDEX].

---

#### 📊 Шаг 2. Тестирование таблиц и бизнес-цепочек (Строго по порядку!)

Поскольку таблицы в СУБД связаны внешними ключами (`Foreign Keys`), создавать записи нужно строго в хронологическом порядке, используя `id` предыдущих объектов [INDEX].

---

### 🏢 Модуль 1: Фермы (`farms`)
* **`POST /api/farms` (Создание фермы)**: Нажмите **"Try it out"**, отправьте JSON. Посмотрите в ответе сгенерированный `id` (будет `1`).
  ```json
  {
    "name": "Южная Мидийная Плантация",
    "region": "Черноморский регион"
  }
  ```
* **`GET /api/farms`**: Вернет массив всех ферм. Каждая ферма автоматически содержит внутри себя вложенный массив своих участков [INDEX].

---

### 🗺️ Модуль 2: Участки / Локации (`locations`)
* **`POST /api/locations` (Создание участка)**: Привязываем участок к созданной ферме через `"farm_id": 1`.
  ```json
  {
    "farm_id": 1,
    "name": "Бухта Тихая",
    "latitude": 44.5561,
    "longitude": 33.5255,
    "depth": 35.5
  }
  ```
  *Запомните ID созданного участка (будет `1`)*.

---

### ⚓ Модуль 3: Канатные линии (`lines`)
* **`POST /api/lines` (Монтаж каната)**: Привязываем канат к участку через `"location_id": 1`. Поле `status` принимает строго: `active`, `inactive` или `maintenance`.
  ```json
  {
    "location_id": 1,
    "name": "Магистраль А-1",
    "depth": 15.0,
    "length": 100.0,
    "status": "active"
  }
  ```
* **`GET /api/lines` / `GET /api/lines/{id}` (Важный аналитический роут)**: Возвращает чистый объект или массив линий. К каждой линии СУБД автоматически прикрепляет три связанных массива: датчики (`sensors`), высаженные партии (`mussel_batches`) и историю технических работ (`operational_events`) [INDEX].

---

### 📦 Модуль 4: Партии мидий (`mussel_batches`)
* **`POST /api/batches` (Высадка урожая)**: Привязываем к линии через `"line_id": 1`. Поле `status` принимает строго: `growing`, `ready`, `lost` или `archived`.
  ```json
  {
    "line_id": 1,
    "batch_number": "BATCH-2026-XYZ",
    "seed_date": "2026-06-20T12:00:00.000Z",
    "initial_count": 10000,
    "status": "growing"
  }
  ```

---

### 🔌 Модуль 5: Регистрация датчиков (`sensors`)
* **`POST /api/sensors` (Установка прибора)**: Связываем строго с канатом через `"line_id": 1`. Поле `sensor_type` принимает: `temperature`, `salinity`, `oxygen`, `turbidity` или `depth`.
  ```json
  {
    "line_id": 1,
    "name": "Термометр глубоководный",
    "sensor_type": "temperature",
    "serial_number": "SN-TEMP-99X",
    "is_active": true
  }
  ```
  *Попробуйте отправить `line_id: 999` — бэкенд заблокирует запрос и выдаст ошибку `404`, защищая целостность СУБД.*

---

### 📈 Модуль 6: Измерения / Телеметрия (`measurements`)
* **`POST /api/measurements` (Лог данных)**: Сюда датчики шлют замеры в море. Привязываем к датчику (`sensor_id: 1`) и канату (`line_id: 1`).
  ```json
  {
    "sensor_id": 1,
    "line_id": 1,
    "parameter": "temperature",
    "value": 18.45,
    "unit": "°C",
    "measured_at": "2026-06-20T14:30:00.000Z"
  }
  ```

---

### 🔬 Модуль 7: Наблюдения биолога (`bio_observations`)
* **`POST /api/observations` (Осмотр урожая)**: Привязываем к линии `1` и партии `1`.
  *Поле `mussel_condition` строго валидирует русский выпадающий список фронта: `отлично`, `хорошо`, `удовлетворительно`, `плохо`, `критично`.*
  *Поле `observed_at` опционально. Если фронт выбрал дату в календаре — бэк запишет её, если поле пустое — бэк сам поставит текущее время.*
  ```json
  {
    "line_id": 1,
    "batch_id": 1,
    "fouling_percent": 12.4,
    "mussel_condition": "отлично",
    "mortality_percent": 0.5,
    "density": 190,
    "avg_size": 45.2,
    "avg_weight": 8.4,
    "pathogen_present": false,
    "observed_at": "2026-06-15T09:00:00.000Z"
  }
  ```

---

### ⚙️ Модуль 8: Операционные события (`operational_events`)
* **`POST /api/events` (Технические работы)**: Запись лога работ (высадка, ремонт, сбор урожая), формирует блок «История» на фронтенде. Поле `created_by` бэкенд заполнит автоматически, вытащив ваш ID из JWT-токена подписи [INDEX].
  ```json
  {
    "line_id": 1,
    "event_type": "repair",
    "description": "Успешно ликвидировали провисание хребтины и добавили 2 поддерживающих буя."
  }
  ```

---

### 🚪 Модуль 9: Безопасный выход (`logout`)
* **`POST /api/auth/logout`**: Отправьте пустой запрос. Бэкенд мгновенно сотрет сессию из таблицы `user_sessions` и занесет текущий JWT-токен в черный список СУБД [INDEX]. Попробуйте после этого сделать любой запрос `GET` — сервер вернет `401 Unauthorized`, подтверждая мгновенную блокировку доступа [INDEX].
